| | |
|---|---|
| ไฟล์หลัก | `strategy17.py` — รันเฉพาะ **M1** (`S17_ALLOWED_TFS`) |
| ประเภท | standalone mean reversion |
| Default | OFF |

> ⚠️ win rate สูง (91-92% backtest) มาจาก **TP สั้น + SL กว้าง** (RR ~0.17) — 1 SL กิน TP ~6 ไม้
> ไม่ใช่เวทมนตร์ ต้องคุม lot เล็กและยอมรับ tail risk

## แนวคิด — 4 ชั้น confluence (เข้าเฉพาะ setup ที่ครบ)

1. **Liquidity Sweep**: แท่ง signal ไส้ทะลุ low/high ของกรอบ `S17_LOOKBACK` (60) แท่ง แต่ **เปิดในกรอบ + ปิดกลับเข้ากรอบ** (stop hunt แล้วถูกปฏิเสธ — เช็ค open ด้วย)
2. **Rejection Wick**: ไส้ฝั่ง sweep ≥ `S17_WICK_MIN_PCT` (30%) ของ range
3. **RSI Extreme**: RSI ≤ 32 (BUY) / ≥ 68 (SELL) ที่แท่ง signal
4. **PD Fibo Zone**: close อยู่ Discount (<38.2%) / Premium (>61.8%) ของกรอบ
+ **Session**: เทรดเฉพาะ Killzones London `14:00–18:00` / NY `19:00–23:00` BKK

## Entry / SL / TP

- **Entry**: LIMIT รอ retrace 61.8% ของแท่ง sweep (`S17_ENTRY_MODE="limit_618"`) — ไม่ fill ใน `S17_LIMIT_CANCEL_BARS` (5) แท่ง → ยกเลิก
- **TP**: entry ± `S17_TP_ATR_MULT` (0.3) × ATR — สั้นมากโดยตั้งใจ
- **SL**: ใต้/เหนือไส้ sweep ∓ `S17_SL_ATR_BUFFER` (1.0) × ATR (buffer ของท่าเอง)
- dedup: 1 ไม้/แท่ง signal + level cooldown `S17_LEVEL_COOLDOWN_BARS` (20) แท่ง (in-memory ไม่ persist)

## Standalone behavior

- bypass Trend Filter (scan) + Sweep Filter block
- skip: Fill Trend Recheck, Pending Trend Check, RSI Recheck (เข้าที่ RSI extreme by design), PD Fibo Plus fill+pending (ใช้ PD ใน detect เอง), Entry Candle, Trail SL, Opposite Order, Limit Guard
- **ไม่เข้า TSO** — ออก lot คงที่ `AUTO_VOLUME` ต่อไม้ (TP สั้นเกินกว่าจะแบ่ง 4 step)
- อยู่ใน `STRONG_TREND_BLOCK_SIDS`
