# ท่าที่ 2 — FVG (Fair Value Gap)

| | |
|---|---|
| ไฟล์หลัก | `strategy2.py` |
| ประเภท | main-flow |
| Default | ON |
| Comment | `{TF}_S2` (normal), `[{TF1}_{TF2}]_S2` (parallel) |

## แนวคิด

- มองหา Fair Value Gap (ช่องว่างราคา) ระหว่างแท่ง `[2]`, `[1]`, `[0]`
- `[1]` ต้องกลืน `[2]` ให้เกิน gap ขั้นต่ำ — ใช้ `engulf_min_price()` (ค่าเดียวกับ engulf ไม่ใช่ค่า hardcode)
- entry คำนวณจาก "ด้านใน" ของ gap
- มี 2 โหมด: **ปกติ (normal)** และ **parallel** ตาม config

## Pattern (signal)

### BUY

- `[1]` เป็นแท่งเขียว และ `Close[1] > High[2] + gap`
- `[0]` ต้อง **ยังไม่ปิด gap** — คือ `Low[0] > High[2]`
- ขนาด gap ต้องไม่เล็กกว่า `engulf_min_price()`
- ถ้า `[0]` เป็น marubozu → เข้าโหมดรอ confirm (ไม่ตั้งทันที)

### SELL — สลับเงื่อนไขตรงข้าม

- `[1]` เป็นแท่งแดง และ `Close[1] < Low[2] - gap`
- `[0]` ยังไม่ปิด gap — คือ `High[0] < Low[2]`

## Sub-pattern ของแท่ง `[0]`

- pattern ที่เปิดใช้งาน: **เขียวกลืนกิน / แดงกลืนกิน** และ **ปฏิเสธราคา**
- `ปฏิเสธราคา` รับได้ทั้งแท่งปิดเขียวและแดง (ไม่มีเงื่อนไขสี) — ใช้ `cancel_bars = 1` (ถ้า limit ไม่ fill ใน 1 แท่งถัดไป → ยกเลิกอัตโนมัติ)
- pattern default `แดง`/`เขียว` (pattern 3-4) **ปิดใช้งาน** แต่ยังคง classification logic ไว้เพื่อ log
- `Limit TP/SL Break Cancel` **ไม่ใช้** กับ pattern 1 (เขียว/แดงกลืนกิน) — skip ผ่าน `c3_type` ใน `pending_order_tf`

## Normal vs Parallel

- **S2 ปกติ**: ต้องเจอ S1 / S2 / S3 ฝั่งเดียวกันย้อนหลังภายใน `S2_NORMAL_CONFIRM_LOOKBACK_BARS` แท่งก่อน จึงจะยอมตั้ง order
- **S2 parallel**: ไม่ใช้กฎยืนยันย้อนหลังนี้ — comment เป็น `[{TF1}_{TF2}]_S2`
- composite TF ของ parallel (เช่น `[M15_H1]`) ไม่มีใน `_swing_data` → fill trend recheck จะ resolve เป็น TF สูงสุด (`[M15_H1] → H1`)

## หมายเหตุ

- เหตุผลย่อยของแท่ง `[0]` (เขียว/แดง/กลืนกิน/ปฏิเสธราคา) ถูกใช้ในข้อความแจ้งเตือน
- FVG ตรวจทั้ง engulf และ gap distance พร้อมกัน
