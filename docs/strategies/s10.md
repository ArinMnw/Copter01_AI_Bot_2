# ท่าที่ 10 — CRT TBS (Candle Range Theory + Three Bar Sweep)

| | |
|---|---|
| ไฟล์หลัก | `strategy10.py` |
| ประเภท | standalone (bypass trend filter) |
| Default | ON (ปุ่ม "เลือกทั้งหมด" ไม่กระทบ — เปิด/ปิดรายตัว) |
| Comment | HTF: `{TF}_S10` · MTF: `[{HTF}-{LTF}]_S10_#{model}` |

## แนวคิด

- รวม Candle Range Theory (CRT) + Three Bar Sweep (TBS)
- หา pattern ที่ราคาทำ liquidity sweep แล้วกลับด้าน
- bypass trend filter (`if sid != 10:` ใน `scanner.py`)

## โหมด

### `CRT_BAR_MODE`
- `2bar` (default): แท่งกลาง sweep + แท่งล่าสุดยืนยัน (TBS compressed)
- `3bar`: มีแท่ง setup เพิ่มก่อน sweep (TBS classic)

### `CRT_ENTRY_MODE`
- `htf` — Entry Model 2 (Confirmation Market)
- `mtf` (default) — Entry Model 3 (LTF Confirmation)

## HTF detect filters (ทุกโหมด)

- TF restriction: HTF mode ใช้ M15+ เท่านั้น; MTF mode ใช้ทุก TF
- Parent range ≥ `CRT_MIN_RANGE_POINTS` (default 200 × points_scale)
- Sweep depth ≥ `CRT_SWEEP_DEPTH_PCT` × parent range (default 10%)
- sweep candle ไม่บังคับสี (BUY/SELL ใช้ได้ทั้งเขียว/แดง ถ้าโครงสร้างผ่าน)

## HTF mode (Entry Model 2)

- ใช้ M15+ เท่านั้น
- เจอ HTF CRT → **market BUY/SELL ทันที** เมื่อ HTF sweep candle ปิดยืนยัน
- entry = HTF sweep_close (2bar) / confirm.close (3bar) · SL = HTF sweep level ± buffer · TP = HTF parent opposite

## MTF mode (Entry Model 3 — default)

หลัง HTF arms → ทุก scan tick (5s) วน LTF rates หา trigger:

```
Phase 1 — Failed-push (pattern confirmation)
  BUY  (HTF sweep low):  LTF RED  + close < HTF parent.low
  SELL (HTF sweep high): LTF GREEN + close > HTF parent.high
Phase 2 — Body engulf 2-bar (entry trigger; concept S1)
  BUY:  prev RED + curr GREEN + curr.close > prev.open
  SELL: prev GREEN + curr RED + curr.close < prev.open
```

Models (คำนวณหลัง engulf):
- **#1 Order Block** (RECOMMENDED) — ย้อนหา bar สีตรงข้าม → entry = OB.open
- **#2 FVG 90%** (concept S2) — 3-bar imbalance, gap ≥ `engulf_min_price()` → entry @ 90% deep ใน gap
- **#3 MSS** — log only (lowest/highest ใน range armed_at→phase1)

Priority: Model 1 → fallback Model 2 → ถ้าทั้งคู่ None ไม่เข้า
`order_mode = "limit"` · SL = `state["sl_target"]` · TP = `state["tp_target"]`

LTF mapping (HTF→LTF): `D1/H12→M15`, `H4→M5`, `H1/M30/M15→M1`

### Continuous re-trigger flow

- Trigger ต้องเจอ **Model 1 AND Model 2** (ไม่ใช่ OR) → fire 2 orders
- หลัง fire → `register_fired_tickets()` เก็บใน `arm_state["fired_tickets"]`, arm **ไม่ถูก consume**
- ticket ปิด: **TP hit** → consume arm | **SL ทั้งคู่** → reset (ถ้า `pre_arm` → `awaiting_choch=True` รอ CHoCH)

## Min SL Distance Guard

- `_sl_distance_ok(direction, entry, sl)` เช็คก่อน return — ใช้ `trade_stops_level + 20pt` (fallback 30pt × points_scale)
- ไม่ผ่าน → return None/WAIT + log `S10_TRIGGER_SKIP_MIN_SL`

## Re-arm guard (กัน duplicate ใน HTF bar เดียว)

- `_last_fired_armed_at[htf_tf]` เก็บ HTF bar window start (unix), helper `_current_htf_bar_start(htf_tf)`
- main scan + pre-arm ใช้ค่าเดียวกัน → guard ครอบทั้ง 2 path

## State

- `_armed_states` persist ผ่าน `bot_state.json` (key `s10_armed_states`)
- `armed_at` = HTF sweep candle open time · expiry = `armed_at + 2 × htf_secs`
- pending MTF ถูกยกเลิกถ้าก่อน fill ราคาไปแตะ HTF parent ฝั่งตรงข้ามแล้ว

## Triple Scale-Out (TSO) — formula พิเศษ S10

- always-4-steps, lot รวม = `base × 4`
- formula: `[min(200pt,TP), min(300pt,TP), TP/2, TP]` ผ่าน `compute_tso_effective_steps(tp, sid="10")`
