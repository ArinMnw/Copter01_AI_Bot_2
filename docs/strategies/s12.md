# ท่าที่ 12 — Range Trading

| | |
|---|---|
| ไฟล์หลัก | `strategy12.py` |
| ประเภท | standalone |
| Default | ON (ปุ่ม "เลือกทั้งหมด" ไม่กระทบ — เปิด/ปิดรายตัว) |
| Comment | `M5_S12_...` |
| sync | `mql5/S12_RangeZone.mq5` |

## แนวคิด

- หา range จาก **pivot swing high/low บน M5** เป็นหลัก → fallback raw high/low ตาม lookback ถ้ายังไม่มี pivot
- แบ่ง range เป็น **buy zone** (ใกล้ swing low) และ **sell zone** (ใกล้ swing high)
- active zone ใช้ **breakout extreme แบบ sticky** ให้สอดคล้องกับ MQL5 — เมื่อราคาทะลุ pivot เดิม zone ใหม่จะค้างจนกว่ามี pivot ใหม่มายืนยัน (ไม่ snap กลับทันที)
- ตั้ง **limit order หลายชั้น** ใน zone เดียวกันได้ (จำนวน = `S12_ORDER_COUNT`)

## Config หลัก

| key | default | ความหมาย |
|---|---|---|
| `S12_ORDER_COUNT` | 3 | จำนวน order ต่อหนึ่งชุด |
| `S12_COOLDOWN_SECS` | 1800 (30 นาที) | cooldown หลัง SL hit |

## Cooldown

- หลัง SL hit → เก็บ `_s12_state["last_sl_time"]` + cleanup ticket ผ่าน `s12_cleanup_tickets()`
- ระหว่าง cooldown S12 จะ **ไม่สร้าง order ใหม่**
- `_s12_scan_status = {"cooldown": "⏳ S12 cooldown N นาที (หลัง SL)"}` เป็นข้อความสถานะ
- **SCAN_SUMMARY จะไม่แสดง S12 block ระหว่าง cooldown** (กัน body ค้างค่า 0.00)

## State

- `_s12_state`: `{"last_sl_time": float}` — **persist** ลง `bot_state.json` (restart แล้วยังจำได้)
- `_s12_scan_status`: dict ข้อความสถานะ scan

## Standalone behavior

- skip: `Limit Guard`, `Limit Sweep` (skip S10/S12/S13), Entry Candle, Trail SL ฯลฯ ของ flow กลาง
