## แนวคิด

นำ concept การเทรดของ TJR (ICT-based) มารวมเป็น 1 ท่า — ไม่เช็คเทรนด์รวมหรือ PD Fibo ตรงกลาง
(standalone) แต่ใช้ HTF bias ของตัวเอง

## ลำดับ Confluence (ต้องผ่านครบทุกชั้น)

1. **Killzone**: เทรดเฉพาะ London/NY (`S18_SESSIONS` default 14:00–18:00, 19:00–23:00 BKK; gate ด้วย `S18_SESSION_FILTER`)
2. **HTF Bias**: เทรดตามทิศ HTF (`S18_HTF_MAP`, บังคับด้วย `S18_REQUIRE_HTF_BIAS`)
3. **Liquidity Sweep**: ไส้กวาด swing low (BUY) / swing high (SELL) แล้วถูกปฏิเสธ (ทิ้งไส้)
4. **MSS (Market Structure Shift)**: หลัง sweep ราคา *close* ทะลุ internal structure ในทิศเดียวกับ bias
5. **Entry Zone**: ราคาอยู่ใน **FVG หรือ Order Block** ในแถบ **OTE 62–79%** (`S18_OTE_LO`/`S18_OTE_HI`); เลือกโซนด้วย `S18_ZONE_PREFER` (FVG/OB), entry edge/mid ด้วย `S18_ENTRY_MODE`
6. **RSI Confirm** (optional): `S18_RSI_FILTER` + `S18_RSI_PERIOD`/`S18_RSI_BUY_MAX`/`S18_RSI_SELL_MIN`
7. **Target & Risk**: RR เป้าหมาย `S18_MIN_RR` / `S18_RR_TARGET`, SL หลังไส้ sweep + `S18_SL_ATR_BUFFER`

## Entry

- **LIMIT** รอ retrace เข้าโซน — ไม่ fill ใน `S18_LIMIT_CANCEL_BARS` แท่ง → cancel
- risk guard: skip setup ที่ risk กว้างเกิน `S18_MAX_RISK_ATR_MULT`

## โครงสร้าง code

- `detect_s18()` (pure — backtest เรียกตรง) + `strategy_18()` wrapper (TF gate + dedup)
- backtest: `sim_s18_backtest.py`

## Standalone behavior

- skip: trend filter, RSI recheck, PD Fibo Plus, entry candle, trail SL, limit guard, PD-zone pending cancel
- **คงไว้**: SL Guard

## State / dedup
