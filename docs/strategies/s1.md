# ท่าที่ 1 — กลืนกิน / ตำหนิ / ย้อนโครงสร้าง

| | |
|---|---|
| ไฟล์หลัก | `strategy1.py` |
| ประเภท | main-flow (เข้า flow กลางครบ: trend filter, recheck, trail, ฯลฯ) |
| Default | ON |
| Comment code | `PA`, `PB`, `PC`, `PE`, `P4` |

## แนวคิด

- ใช้แท่ง `[2]`, `[1]`, `[0]` (เก่า→ใหม่) เป็นแกนหา pattern กลืนกิน/ตำหนิ
- มีทั้งฝั่ง BUY และ SELL (สลับสีตรงข้ามกัน)
- ใช้ `engulf_min_price()` เป็นระยะขั้นต่ำของคำว่า "กลืนกิน" (gap) — scale ตาม instrument
- body ขั้นต่ำของแท่งสัญญาณ = `35%`
- รองรับ **Zone Mode** (`S1_ZONE_MODE = "zone"`) และ **Forward Confirm** เพิ่มเติม

## Pattern

### Pattern A — กลืนกินต่อเนื่อง 2 แท่ง

BUY:
- `[2]` แดง
- `[1]` เขียว และ `Close[1] > High[2] + gap`
- `[0]` เขียว และ `Close[0] > High[1] + gap`
- body ของ `[1]` ≥ 35%

SELL: สลับสีตรงข้าม
- `[2]` เขียว
- `[1]` แดง และ `Close[1] < Low[2] - gap`
- `[0]` แดง และ `Close[0] < Low[1] - gap`
- body ของ `[1]` ≥ 35%

### Pattern B — ตำหนิ + กลืนกลับ

BUY:
- `[2]` แดง
- `[1]` เขียวแบบ "ตำหนิ" (ไส้/ช่วงแท่งกินเข้าไปใน zone ของ `[2]`)
- body `[1]` ≥ 35%
- `[0]` เขียวกลืน `[1]`

SELL: สลับสีตรงข้าม

### Pattern C — ย้อนโครงสร้าง 3 แท่ง

- ชุดย้อนโครงสร้าง 3 แท่ง (ไม่ใช่กลืนกินตรง ๆ แบบ A)
- ใช้ logic เฉพาะใน `strategy1.py`

### Pattern E — แดง-แดง-เขียวกลืน (และสลับ)

BUY:
- `[2]` แดง, `[1]` แดง, `[0]` เขียว
- `Close[0] > High[1] + gap`
- body `[0]` ≥ 35%

SELL: สลับสีตรงข้าม
- `[2]` เขียว, `[1]` เขียว, `[0]` แดง
- `Close[0] < Low[1] - gap`
- body `[0]` ≥ 35%

### Pattern 4 แท่ง (`P4`)

- pattern ย่อยอีกแบบในท่า 1 (ใช้ 4 แท่ง) — รายละเอียดตาม logic ใน `strategy1.py`

## Zone Mode (`S1_ZONE_MODE = "zone"`)

- ขั้น detect จะ **ไม่ block** setup เพราะหลุด zone — ตั้ง order ไปก่อน แล้วประเมินทีหลัง
- การเช็ก zone ใช้ `min low` (BUY) / `max high` (SELL) ของ **ทุกแท่งใน pattern** เทียบ Swing Low/High (ไม่ใช่แค่แท่ง `[1]`)
- pending อยู่นอก zone → ยกเลิก limit
- pending ยังอยู่ใน zone → คง order
- fill แล้วอยู่นอก zone และ `profit < 0` → ปิด position
- fill แล้วอยู่นอก zone แต่ `profit >= 0` → ไม่ปิด
- metadata แนบกับ setup ใน `s1_zone_meta`

## Forward Confirm

- S1 ตั้ง order ไปก่อน แล้วรอดู S2 หรือ S3 ฝั่งเดียวกันใน TF เดียวกันภายใน **5 แท่งข้างหน้า**
- ยัง pending + ครบ 5 แท่งไม่เจอ → ยกเลิก pending
- fill แล้ว + ครบ 5 แท่งไม่เจอ → ปิด position
- เจอแล้ว (ไม่ว่า pending หรือ fill) → ไม่ทำอะไรต่อ

## Entry / SL / TP

- entry/SL/TP คำนวณจาก pattern + swing (ดู `entry_calculator.py` / `tp_sl/strategy1_tp_sl.py`)
- TP พยายามหา swing TP ที่เหมาะสม → ถ้าไม่เจอใช้ fallback **RR 1:1**
