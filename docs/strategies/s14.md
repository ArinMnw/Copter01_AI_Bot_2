# ท่าที่ 14 — Sweep RSI

| | |
|---|---|
| ไฟล์หลัก | `strategy14.py` |
| ประเภท | standalone (market order ทันที) |
| Default | OFF |
| Comment | `{TF}_S14_engulf` / `{TF}_S14_sweep` |

## แนวคิด

- ใช้ RSI หา reversal zone (LL/HH ของ RSI ในช่วง reversal bars)
- ตรวจ pattern บนแท่งล่าสุดว่ามี **Engulf** หรือ **Sweep** ทะลุ zone นั้น
- เจอ pattern → เปิด **market order ทันที** (ไม่มี pending)

## Config

| key | default | ความหมาย |
|---|---|---|
| `S14_RSI_PERIOD` | 14 | RSI period |
| `S14_RSI_APPLIED_PRICE` | `close` | applied price |
| `S14_REVERSAL_LOOKBACK` | 50 | bars ย้อนหา reversal zone |
| `S14_ENGULF` | True | เปิด sub-pattern Engulf |
| `S14_SWEEP` | True | เปิด sub-pattern Sweep |
| `S14_LL_USE_HHLL` | False | ใช้ HHLL HL/HH เป็น ref เพิ่ม |
| `S14_FLIP_ENABLED` | True | ปิดฝั่งตรงข้ามอัตโนมัติก่อนเปิดใหม่ |

## Sub-pattern

- **Engulf** — แท่งล่าสุด close ทะลุ LL zone (BUY) / HH zone (SELL)
- **Sweep** — ไส้แท่งล่าสุดทะลุ zone แต่แท่งปิดกลับเข้ามา (rejection wick)

## Signal

BUY: หา `LL zone` ย้อนหลัง → Engulf `close[0] < ll_val` หรือ Sweep `low[0] < ll_val` + close กลับเหนือ
SELL: หา `HH zone` ย้อนหลัง → Engulf `close[0] > hh_val` หรือ Sweep `high[0] > hh_val` + close กลับต่ำกว่า

## Order flow (Flip logic)

- เปิด **market order** ทันที
- ถ้า `S14_FLIP_ENABLED = True` → เรียก `_clear_opposite_s14_exposure()` ปิด S14 ฝั่งตรงข้ามบน TF เดียวกันก่อน
  - Telegram: `↔️ [TF] S14 Flip — ปิดฝั่ง BUY/SELL → เปิด SELL/BUY แทน`
  - log: `S14_REVERSE_CLOSE` / `S14_REVERSE_CLOSE_FAIL`

## Standalone behavior

- **bypass Trend Filter** (scan): `sid not in (9, 10, 13, 14)`
- **bypass RSI Fill Recheck**: `sid in (1, 9, 11, 14)`
- **bypass Fill Trend Recheck**: `sid in (9, 10, 14)`
- ไม่เข้า: Entry Candle, Trail SL, Opposite Order, Limit Guard, SL Guard
- **ใช้** PD Fibo Plus filter ใน scan loop (entry ต้องอยู่ Discount `<38.2%` / Premium `>61.8%`)

## Triple Scale-Out (TSO)

- ใช้ TSO ผ่าน `open_order_market()` (ตั้งแต่ 2026-05-26) — scale lot ×4 ถ้า `SCALE_OUT_ENABLED`
- `_scale_out_register_ticket()` register พร้อม `is_pending=True` (trailing.py update entry จาก fill price จริง)
