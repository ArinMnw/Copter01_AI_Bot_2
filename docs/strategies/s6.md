# ท่าที่ 6 / 7 — S6 (Trail ท่า 2/3) + S6i (Swing อิสระ)

| | |
|---|---|
| ไฟล์หลัก | `trailing.py` (`_s6_process_ticket`, `_s6i_process_ticket`, `check_s6_trail`) |
| ประเภท | position management (ไม่ใช่ท่าเข้า order แบบ pattern) |
| รหัส | `6` = S6, `7` = S6i |
| State | `_s6_state`, `_s6i_state` (module-level ใน `trailing.py`) |

## ภาพรวม

S6 และ S6i **ไม่ใช่** ท่าหา pattern เข้า order ใหม่ตรง ๆ — เป็น **state machine จัดการ position**
ที่ถือไว้แล้ว โดยคอยดูสัญญาณท่า 1 ฝั่งตรงข้ามเพื่อปรับ TP / ปิด / ตั้งไม้สวน

แกนกลางใช้ฟังก์ชันร่วมกัน `_s6_process_ticket(...)` (`mode_tag` = `"S6"` หรือ `"S6i"`)

## S6 — Trail ของ position ท่า 2/3

- ทำงานต่อจาก position ที่มาจากท่า 2 / 3
- ทุกรอบ scan จะรัน `strategy_1(rates)` บน TF ของ position
- ถ้าเจอสัญญาณท่า 1 **ฝั่งตรงข้าม** กับ position:
  - ในเฟส `wait`: ตั้ง **TP = entry ของท่า 1 ฝั่งตรงข้าม** (ถ้า TP นั้น valid กับฝั่ง position)
  - บันทึก `tp_set_by_s1` ใน state แล้วแจ้ง Telegram `🎯 S6 ตั้ง TP = ท่า1 Entry`
- มี phase machine (`wait` → นับ/จัดการต่อ) เก็บใน `_s6_state[ticket]`

## S6i — 2 High 2 Low อิสระ

- เป็น swing logic อิสระ (independent) — ตั้ง order ใหม่ได้ภายใต้เงื่อนไขของ swing + pattern ตรงข้าม
- ใช้ helper เช็ค S1/S3 pattern แบบ **ไม่ผ่าน zone filter** (`_s6i_process_ticket`)
- init: เก็บ `swing_ref` + เช็ค S1 + ตั้ง TP เริ่มต้น (`s6i_init_tp`) ที่ opposite entry
- เมื่อมี swing ใหม่ → ปรับ TP (`s6i_new_swing_tp`)
- ถ้าไม้ฝั่งตรงข้าม (opposite limit) ถูก fill → ปิด position เดิม (`S6i: opposite limit filled`)
- comment ของ S6i order: `..._S6i_buy` / `..._S6i_sell`

## Swing source (ทั้ง S6/S6i)

- `_get_s6_prev_swing_high(rates, lookback=100, tf=...)` / `_get_s6_prev_swing_low(...)`
- ใช้ **HHLL swing ก่อน** → fallback pivot/simple ถ้า HHLL ไม่มีข้อมูล

## หมายเหตุ

- ทั้งคู่ทำ audit ทุกครั้งที่แก้ SL/TP (`_audit_sltp_event` + `_notify_sltp_audit`) → log ลง `logs/debug/sltp_audit.log`
- รายละเอียด state/phase ที่แม่นยำที่สุดอยู่ใน `trailing.py` (`_s6_process_ticket` ~บรรทัด 5861, `_s6i_process_ticket` ~6117)
- เนื่องจากเป็น state machine ที่ผูกกับ lifecycle ของ position — เวลาแก้ ให้ไล่ `position_tf → position_sid → state_dict[ticket]` เสมอ
