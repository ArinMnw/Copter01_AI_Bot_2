### รายละเอียดพฤติกรรมราคา (Weekly Detailed Cases)
- **[สัปดาห์ที่ 1]:** 
  * **เคสที่ 1 (WIN - TF M1):** ราคาวิ่งขึ้นทดสอบโซน SP (Supply Zone) อย่างรุนแรงจนเกิดการทะลุหลอก (Fakeout) และทิ้งไส้ยาว (Rejection) กลับลงมาปิดต่ำกว่าโซน DM ล่าสุด ระบบจึงจับโครงสร้าง `SP_Trap` ตั้ง SELL LIMIT ดักไว้ที่ตำหนิราคาบน ราคาร่วงทะลุเป้า TP อย่างรวดเร็ว
  * **เคสที่ 2 (LOSS - TF M5):** ราคาหลุดทรงทะลุแนว DM แต่ปิดด้วยแท่งเต็มที่ไม่ทิ้งไส้ ส่งผลให้ไม่เกิด Rejection และโดนแรงขายกดทับ โดนทุบแตกขาดทุนตามจุด Stop Loss ปกติ
- **[สัปดาห์ที่ 2]:** 
  * **เคสที่ 1 (WIN - TF M15):** ตลาด Sideway สะสมสภาพคล่อง ข่าวทำราคาพุ่งกวาด High เดิม (Liquidity Sweep) ก่อนม้วนกลับเข้า FVG อย่างรวดเร็ว เกิดรูปแบบกวาดไส้ ระบบยิงออเดอร์ในจังหวะ Sniper Entry ทำกำไรจนจบเทรนด์
  * **เคสที่ 2 (LOSS - TF M30):** ตลาดผันผวนสูง อินดิเคเตอร์ส่งสัญญาณ False Breakout หลอกกิน Stop Loss ทั้งฝั่ง Buy และ Sell
- **[สัปดาห์ที่ 3]:** 
  * **เคสที่ 1 (WIN - TF H1):** ตรงตามภาพหน้าจอใน PDF! กราฟสร้างฐานสะสมกำลัง เกิดแท่งกลืนกินเบรคเอ้าท์และลงมา Retest ที่ระดับ 50-60% ของการตั้งค่า Fibo แล้ววิ่งทะยานขึ้น
  * **เคสที่ 2 (LOSS - TF H4):** โครงสร้างระดับ Timeframe ใหญ่ถูกทำลายด้วยข่าวแดง ราคาวิ่งสวนทางเทคนิคคอลทะลุ SL ทันที
- **[สัปดาห์ที่ 4]:** 
  * **เคสที่ 1 (WIN - TF M5):** จังหวะปิดเดือน ตลาดกระชากกินไส้ (Fakeout) กลับสู่ทิศทางหลัก เข้าเคส `DM_Trap` ราคาเปิดกระโดดกินกำไรสวยงาม
  * **เคสที่ 2 (No-Trade Case):** มีสภาวะตลาดหลอกคล้าย SP_Trap แต่ไม่ทิ้งไส้ Rejection ชัดเจน ระบบ "Skip" ไม่เปิดออเดอร์ ช่วยปกป้องทุน 100%

### Log Matrix Architecture
1. **การรับพารามิเตอร์:** ใช้ `argparse` รองรับ `--days` แบบ Array (30, 60, 90, 120, 180) และ `--tf` เพื่อแบ่งสแกนแต่ละกรอบเวลา
2. **การคำนวณ Net P&L (Win/Loss):** คำนวณแบบจำลองตาม Contract Size ($100), Fixed Lot (0.1) และหักลบค่า Commission / Spread ออกจากผลกำไรจริงทุกออเดอร์ เพื่อความแม่นยำ 
3. **การคัดกรอง (No-Trade Case):** มีเงื่อนไขดักจับ False Signal จากการปิดแท่งที่ไม่มีลักษณะ Wick Rejection ที่ชัดเจนตามตำรา เพื่อเลี่ยงการเข้าเทรดสุ่มเสี่ยง
