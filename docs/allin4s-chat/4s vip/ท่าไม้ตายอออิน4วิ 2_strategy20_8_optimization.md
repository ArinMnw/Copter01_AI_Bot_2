# Strategy 20.8 Optimization Report (ท่าไม้ตายอออิน4วิ 2)

## 1. การวิเคราะห์จุดบกพร่อง (Gap Analysis)
จากการตรวจทานโค้ดเดิม `strategy20_8.py` เทียบกับคัมภีร์ `ท่าไม้ตายอออิน4วิ 2_pdf.md` พบข้อจำกัดดังนี้:
1. **Liquidity Sweep Constraint:** โค้ดเดิมบังคับไม่ให้ราคาหลุด Low/High เดิมเลย (`_low >= prev_low - 0.5`) ซึ่งพลาดจังหวะการทำ Stop Hunt ที่สถาบันตั้งใจทุบให้หลุดแล้วดึงกลับ (Failure to Close)
2. **Fixed Risk-Reward:** มีการ Hardcode ค่า TP ไว้ที่ 700 จุด และ SL 100 จุด ซึ่งในภาวะตลาดที่ความผันผวนเปลี่ยนไป จะทำให้ RR เพี้ยนและเสี่ยงต่อการโดนสะบัดชน SL ง่าย
3. **ขาดตัวกรองการอัดฉีดออเดอร์สวน (Institutional Momentum Filter):** หากมีแท่งเทียนยาวผิดปกติสวนทางลงมา การเข้าไปสวนเพื่อหวัง Rejection อาจจะเป็นการรับมีดที่ตกลงมา

## 2. การอัปเกรดเชิงลึก (Quantitative Upgrades)
- **Dynamic Risk Management:** ปรับมาใช้ `Dynamic ATR` ในการคำนวณ SL และ TP โดยกำหนดให้ SL และ TP ยืดหยุ่นตามสภาวะตลาด และล็อกเป้า RR ไว้ที่ขั้นต่ำ 1:1.5 เพื่อให้เกิดผลกำไรแบบ Compounding อย่างยั่งยืน
- **Institutional Displacement Block:** เพิ่มตัวกรองเช็กแท่งก่อนหน้า หากมีความยาวเกิน 2 เท่าของ ATR จะทำการ Block สัญญาณทันที เพื่อหลีกเลี่ยงการถูกทุบทะลุโซน
- **Liquidity Sweep Rule:** อนุญาตให้แท่งเทียนตวัดไส้แทงทะลุโครงสร้าง 2L/2H เดิมได้เพื่อกวาด SL แต่เพิ่มเงื่อนไขว่า **"ราคาปิด (Close) จะต้องดึงกลับมาปิดเหนือ/ใต้ฐานเดิม (Failure to Close)"** ซึ่งเป็นพฤติกรรมดักกินรายย่อยที่แท้จริง
