## ส่วนที่ 1: คู่มือหน้าเทรดท่าย่อยและโค้ดฉบับสมบูรณ์ (100% Comprehensive Strategy & Code Blueprint)

1. **ชื่อและแนวคิดท่าย่อย**: 
   - **S20.6 FVG Retest Models (Standalone)**
   - ท่าย่อยนี้ถูกแยกออกมาเพื่อให้การทำงานของการเข้าออเดอร์ในรูปแบบ FVG เป็นอิสระ โดยมุ่งเน้นสภาวะตลาดที่มีการเกิด Imbalance อย่างรุนแรงจนทิ้ง GAP เอาไว้ และรอจังหวะที่ราคาย้อนกลับมาเพื่อเติมเต็มช่องว่าง (Mitigation) 

2. **เงื่อนไขและสัญญาณคอนเฟิร์มระบบ (Setup & Confirmation)**: 
   - ค้นหาพฤติกรรมแท่งเทียน 3 แท่งที่สร้าง FVG (แท่ง 1 และแท่ง 3 ไม่สัมผัสกัน)
   - หากแท่งเทียนไม่ได้ทำ High/Low ใหม่ แสดงว่าแนวโน้มตลาดกำลังจะย้อนกลับมา Sweep รับของที่ FVG ปัจจุบัน
   - **Commitment Check**: แท่งเทียนที่แตะขอบ FVG (Retest) จะต้องแสดงถึงการกลับตัวอย่างชัดเจน เช่น ปิดด้วยรูปแบบ Engulfing (กลืนกิน) หรือมีไส้เทียนยาวเป็น Pin Bar (ไส้ยาวกว่าเนื้อ 2 เท่า)
   - **Trap Validation**: หากแท่งเทียนแตะ FVG แล้วปิดในลักษณะตัน (ไม่มีไส้เทียนในทิศทางที่จะไปต่อ) ถือเป็นสภาวะ Trap ห้ามเข้าเทรด เพราะราคามีแนวโน้มที่จะ Sweep ลงไปโซนลึกกว่า

3. **ตรรกะเทคนิคอลและโค้ดเงื่อนไขระบบ (Technical Logic & Source Code)**: 
   - พัฒนาใน `strategy/s20.6/strategy20_6.py` โดยเขียนฟังก์ชัน `_find_fvg_retest_models` เพื่อย้อนเช็คการเกิด FVG ภายใน 40 แท่งเทียนย้อนหลัง และตรวจสอบการเคลื่อนที่ของราคาก่อนหน้า ว่าปิดทะลุ GAP ไปแล้วหรือไม่
   - คำนวณ Entry, SL และ TP โดยใช้ ATR (14) ควบคู่กับอัตราส่วน Fibonacci Extension เพื่อวางเป้าหมายกำไร

4. **แผนพฤติกรรมราคาและการเลือกทาง (Price Action Scenarios)**: 
   - เมื่อเข้าเงื่อนไข จะส่ง Market Order (ตีความจากการปิดแท่งเทียนสัญญาณ)
   - ถ้าราคายืนเหนือขอบ GAP ได้ จะวิ่งไปสู่ TP ที่วัดจาก Fibonacci Expansion ของรอบสวิง
   - ถ้าราคาทำลายขอบ GAP และปิดหลุดลงไป จะโดน Stop Loss ที่สวิงโลว์ล่าสุด (หรือ ATR Buffer) ทันที

5. **ข้อควรระวังและคำเตือน (Warnings)**: 
   - กรณีมีแท่งพุ่งแรง (Strong Trend) ห้ามเข้าสวน FVG ทันที ยกเว้นจะเกิดแท่งพักตัวหรือ Doji ก่อน
   - ถ้าเกิด Doji ลอยๆ โดยไม่ถึงขอบ FVG จะถือเป็น Indecision รอแท่งถัดไปยืนยัน

6. **ระบบ Config, Telegram UI, Bypass และ Scanner**:

## ส่วนที่ 3: รายละเอียดพฤติกรรมราคาผลจำลอง Backtest แบบรายสัปดาห์ (Weekly Detailed Backtest Cases)

- **[สัปดาห์ที่ 1]:** 
  * เคสที่ 1 (WIN): ใน TF M15 ราคาเกิดโครงสร้าง Bearish BOS พร้อมทิ้ง FVG สีแดงไว้เบื้องหลัง ราคาดีดกลับขึ้นมาทดสอบโซน (Premium FVG) เกิดเป็นแท่ง Pin Bar สีแดงหางยาว (Reject ชัดเจน) ระบบสั่งเปิด SELL ทันที ราคาทิ้งดิ่งลงต่อเนื่องจนถึงเป้าหมายกำไรภายใน 3 แท่งเทียน
  * เคสที่ 2 (LOSS): ใน TF M5 ราคาทำ FVG ฝั่ง BUY และราคาพักตัวลงมา แต่ก่อนจะถึง GAP ราคาเกิดสภาวะบีบตัวและปิดแท่งด้วยอารมณ์คลุมเครือ ไม่มีไส้ล่าง (Trap) ระบบเผลอเข้าออเดอร์ตามเงื่อนไขเนื้อเทียน (False Signal) ผลคือราคากระชากลงมากวาด Liquidity ใต้ FVG ทันที ทำให้ชน Stop loss
- **[สัปดาห์ที่ 2]:** 
  * เคสที่ 1 (WIN): ใน TF H1 ข่าวการตัวเลขเศรษฐกิจประกาศออกมา ราคาเกิด Imbalance อย่างรุนแรงจนมีช่องว่าง FVG กว้างมาก เมื่อตลาดสงบ ราคาค่อยๆ ซึมลงมาแตะยอด FVG และเกิดลักษณะแท่ง Bullish Engulfing ระบบเข้า BUY ตาม Commitment ที่ยืนยันได้และวิ่งยาวกลับไปทำ High ใหม่
  * เคสที่ 2 (LOSS): ใน TF M1 ราคามีความผันผวนสูง (Noise) เจอ FVG ถี่มาก ระบบยิงออเดอร์สวนเทรนด์ย่อยที่สวิงไปมา ผลคือโดน Stop Loss ปลายไส้ไปหลายออเดอร์ก่อนที่ราคาจะเลือกทาง (Whipsaw effect)
- **[สัปดาห์ที่ 3]:** 
  * เคสที่ 1 (WIN): TF H4 การวิ่งที่เป็นรอบสวิงใหญ่ (Swing Trading) เกิด FVG ขึ้นอย่างชัดเจน ราคาลงมาเหยียบ GAP ด้วยรูปแบบแท่งเทียน Doji สลับกับ Pin bar หางยาว (Indecision to Reversal) ออเดอร์ใน H4 นี้นำพากำไรสูงมากด้วย Lot ขนาดกลางและระยะ R:R กว้าง
  * เคสที่ 2 (LOSS): การขยายระยะ Stop Loss ตาม ATR 1.5 บางครั้งเมื่อเจอแท่งเบรคหลอก (Fakeout) ที่ยาวมากๆ ก็ทำให้เงินสูญเสียมากกว่าปกติตามความผันผวนของแท่ง
- **[สัปดาห์ที่ 4]:** 
  * เคสที่ 1 (WIN/LOSS): ในช่วงปลายเดือน พฤติกรรมราคาชะลอตัวและบีบเป็นสามเหลี่ยม (Consolidation) FVG ที่เกิดในกรอบ Whipsaw ล้วนใช้งานไม่ได้ ส่งผลให้ระบบใน TF เล็กๆ เสียหายต่อเนื่อง แต่ออเดอร์บน TF H1 ยังสามารถรักษาสมดุลไว้ได้เพราะไม่มีพฤติกรรม Trap 
  * เคสที่ 2 (No-Trade Case): TF M30 ราคาเกิด FVG ขาขึ้น แต่เทรนด์ภาพรวมโครงสร้าง (HHLL) เป็น "Strong Bear" ระบบจึงเปิดโหมด *Trend Filter Bypass* ทำให้ออเดอร์ BUY นี้ไม่ถูกประมวลผล ถือเป็นการรักษาเงินทุนที่มีประสิทธิภาพ เพราะหลังจากนั้น 2 แท่งเทียนราคาก็ทุบลงแรงทำ Lower Low

## ส่วนที่ 4: ตรรกะการคำนวณเคสและระบบแสดงผล (Log Matrix Architecture)

1. **ตรรกะการรับพารามิเตอร์**: สคริปต์ `backtest_S20_6_runner_mt5.py` ใช้ `argparse` ในการรับข้อมูล Command Line เช่น `--days 30 --tf all` เพื่อวนลูปประมวลผลข้อมูลแยกทีละ Timeframe โดยใช้ไลบรารี `datetime` ในการบวกลบวันและจัดการ Timezone ให้เป็น BKK ก่อนดึง API MT5
2. **การคำนวณ P&L (Win)**: หากราคาของแท่งเทียนถัดๆ ไปวิ่งไปแตะ TP (ตรวจสอบด้วยเงื่อนไข `high >= tp` สำหรับ BUY หรือ `low <= tp` สำหรับ SELL) ระบบจะคำนวณกำไร = `(ระยะทาง * Contract Size * Volume) - (Commission * Volume)`
3. **การคำนวณ P&L (Loss)**: หากแท่งเทียนวิ่งสวนมาแตะ SL (ตรวจสอบด้วยเงื่อนไข `low <= sl` สำหรับ BUY หรือ `high >= sl` สำหรับ SELL) ระบบจะหักลบเงินขาดทุน = `(ระยะทาง * Contract Size * Volume) - (Commission * Volume)`
4. **ตรรกะ No-Trade**: ในฟังก์ชันการหา FVG `_find_fvg_retest_models` หากตรวจพบว่าแท่งปิดมีลักษณะ Trap (มีเนื้อตัน ไส้น้อยกว่า ATR * 0.02) ฟังก์ชันจะ return การ Block ออกมาทันทีเพื่อเป็น No-Trade Case ป้องกัน Fakeout
