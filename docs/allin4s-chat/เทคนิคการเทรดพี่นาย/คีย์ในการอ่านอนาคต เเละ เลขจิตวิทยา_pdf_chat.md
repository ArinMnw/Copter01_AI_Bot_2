## 2. การวิเคราะห์กลไกตลาดและโครงสร้างราคา (Institutional Price Action)
- **ประเด็นหลัก**: การวิเคราะห์ความขัดแย้งของเทรนด์ใหญ่และเทรนด์ย่อย (Timeframe D1 เทียบกับ H12)
- **ทิศทางและการล่า Liquidity**: 
  - หาก Day 1 ปิดแดง แต่แท่ง H12 ล่าสุดปิดเขียว (ทิ้งไส้ล่างยาว) แสดงถึงแรงซื้อที่พยายามดึงราคากลับ (Liquidity Hunt) เป็นสัญญาณซื้อ (BUY)
  - หาก Day 1 ปิดเขียว แต่แท่ง H12 ล่าสุดปิดแดง (ทิ้งไส้บนยาว) แสดงถึงแรงขายที่กดดันราคา เป็นสัญญาณขาย (SELL)
- **จุดเข้า (Entry)**: เข้าออเดอร์เมื่อกราฟ M5 ย่อลงมาที่ปลายไส้ของ H12 ในวันก่อนหน้า 
- **การตั้ง TP/SL**: ใช้ความรู้จากเลขจิตวิทยา (Psychological Numbers) โดยเป้าหมาย TP ให้ลงท้ายด้วยเลข 7 (เช่น xxxx7.00) เพราะมักจะเป็นจุดที่ราคาไปถึงเสมอ

## 3. การออกแบบระบบเทรดและตรรกะเงื่อนไข
- **ไฟล์สร้างกลยุทธ์**: `strategy/s20.12/strategy20_12.py`
- **ตรรกะเงื่อนไข (Logic Check)**:
  - ดึงข้อมูล D1 แท่งก่อนหน้าและ H12 แท่งสุดท้ายของเมื่อวาน
  - `BUY Signal`: D1 สีแดง + H12 สีเขียว + H12 มีไส้ล่าง (Bottom Wick) กราฟปัจจุบันอยู่แถวปลายไส้ของ H12
  - `SELL Signal`: D1 สีเขียว + H12 สีแดง + H12 มีไส้บน (Top Wick) กราฟปัจจุบันอยู่แถวปลายไส้ของ H12
  - คำนวณ Take Profit ให้ตกที่ตัวเลขทางจิตวิทยาที่ลงท้ายด้วย 7 (VIP Rule)

## 4. การจัดการ Telegram UI และ Config (System Integration)
- **config.py**: จัดการเพิ่มตัวแปร Global `S20_12_ENABLED`, `S20_12_TF_ENABLED`, `S20_12_COMPOUNDING_ENABLED`, `S20_12_RISK_PCT` และ `S20_12_MAX_LOT` เพื่อเตรียมพร้อมรองรับระบบทบต้น
- **handlers/keyboard.py**: สร้างปุ่มเมนูสำหรับเปิด/ปิด S20.12 และปรับการตั้งค่า Timeframe รวมถึง Risk % 
- **handlers/callback_handler.py & handlers/text_handler.py**: ดักจับ Event สำหรับรับค่าจากผู้ใช้ แล้วเซฟลง `bot_state.json` ได้อย่างสมบูรณ์
- **scanner.py**: ติดตั้งระบบรันคำสั่งลงในลูปทำงานหลัก เพื่อให้บอทเริ่มค้นหาสัญญาณทันทีที่ผู้ใช้กดเปิดใช้งานจาก Telegram
