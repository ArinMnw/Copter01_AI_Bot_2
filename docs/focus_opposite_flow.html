Focus Opposite — Flow Diagram (frozen_side markers — แยกต่อฟีเจอร์)

กติกาใหม่: ใช้ marker ฝั่งที่ถูก freeze แยก คนละตัว ต่อฟีเจอร์ (Trail SL / ECM) ทุกไม้ในฝั่งที่ตรงกับ marker freeze ตลอด ส่วนไม้ฝั่งตรงข้ามจะ trail/ECM ได้ ก็ต่อเมื่อ ฝั่ง frozen มี position ที่ กำไร > threshold และ TF ผ่านเกณฑ์ (ของฟีเจอร์นั้น ๆ)

สรุปพฤติกรรม
  • Focus Opposite แยกต่อฟีเจอร์ — Trail SL และ Entry Candle Mode ใช้ config คนละชุดและ marker คนละตัว
  • มี marker 2 ตัว: trail_sl_frozen_side และ entry_candle_frozen_side (ค่า BUY/SELL/None)
  • ทั้งสอง marker ถาวรใน bot_state.json — restart แล้วไม่หาย
  • แต่ละ marker ใช้ config ของฟีเจอร์ตัวเอง (enabled / points / tf_mode)
  • toggle OFF→ON ของฟีเจอร์ไหน รีเซ็ต marker ของฟีเจอร์นั้นเท่านั้น — ฟีเจอร์อีกตัวไม่ได้รับผลกระทบ
  • ฝั่งที่ตรงกับ marker ของฟีเจอร์นั้น → freeze เสมอ (ไม่สนใจ threshold/TF)
  • ฝั่งตรงข้าม marker → trail/ECM ได้ก็ต่อเมื่อ position ฝั่ง frozen มี กำไร > threshold และ TF ผ่านเกณฑ์ (ตาม config ของฟีเจอร์นั้น)

การตั้งค่า marker (แยกตามฟีเจอร์)

ตารางนี้ใช้ได้กับทั้ง trail_sl_frozen_side และ entry_candle_frozen_side — logic เหมือนกันทุกประการ ต่างกันแค่ว่าแต่ละ marker ดูเฉพาะ enabled/threshold/TF ของฟีเจอร์ตัวเอง

เหตุการณ์พฤติกรรมของ marker
ไม่มี order ทั้งสองฝั่ง แล้วเปิด BUY เป็นไม้แรก marker = BUY
ไม่มี order ทั้งสองฝั่ง แล้วเปิด SELL เป็นไม้แรก marker = SELL
มี BUY อยู่แล้ว → เพิ่ม SELL ภายหลัง marker คงเดิม = BUY
BUY เก่า + SELL เก่า + BUY ใหม่เข้ามา marker คงค่าตามของเดิม (ฝั่งที่ถูก freeze ครั้งแรก)
ไม่มี position และไม่มี pending limit ทั้งสองฝั่ง reset marker ของทั้งสองฟีเจอร์ → None
ผู้ใช้ toggle Focus Opposite OFF → ON ที่ฟีเจอร์ Trail SL reset เฉพาะ trail_sl_frozen_side → None (ECM ไม่โดน)
ผู้ใช้ toggle Focus Opposite OFF → ON ที่ฟีเจอร์ ECM reset เฉพาะ entry_candle_frozen_side → None (Trail SL ไม่โดน)
"order ที่ถือฝั่งเดียว" นับรวม position + pending limit/stop ทั้งคู่ ถ้ามีอย่างน้อย 1 รายการฝั่งใดฝั่งหนึ่ง โดยฝั่งตรงข้ามไม่มีอะไรเลย ถือว่าฝั่งนั้นเป็น frozen_side

เงื่อนไขแบบตาราง

#เงื่อนไขใช้กับฝั่งไหนรายละเอียด
1 Master toggle ทั้งสองฝั่ง TRAIL_SL_FOCUS_NEW_ENABLED (สำหรับ Trail SL)
ENTRY_CANDLE_FOCUS_NEW_ENABLED (สำหรับ ECM)
2 position ฝั่งตัวเอง = frozen_side ฝั่ง frozen freeze เสมอ — ไม่ตรวจ threshold, ไม่ตรวจ TF, ไม่สนใจมี opposite หรือไม่
3 กำไรของ position ฝั่ง frozen เกิน threshold ฝั่งตรงข้าม (gate) ถ้า frozen_side = BUY: ตรวจ bid_cur − buy.price_open > threshold ของ BUY อย่างน้อย 1 ไม้
ถ้า frozen_side = SELL: ตรวจ sell.price_open − ask_cur > threshold ของ SELL อย่างน้อย 1 ไม้
threshold = POINTS × point + spread
4 TF mode ฝั่งตรงข้าม (gate) combined: ขอแค่มี position ฝั่ง frozen ที่ผ่าน threshold ก็พอ
separate: position ฝั่ง frozen ที่ผ่าน threshold ต้องมี TF ตรงกับไม้ที่กำลังเช็ก
ฝั่ง frozen: freeze เสมอ ไม่มีเงื่อนไขปลดล็อก
ฝั่งตรงข้าม: ถ้าข้อ 3 + 4 ผ่าน → trail/ECM ปกติ / ถ้าไม่ผ่าน → freeze (รอจน frozen ฝั่งทำกำไรได้)
Gate ปลดล็อก = เช็ก metrics ของ ฝั่ง frozen (ไม่ใช่ของไม้ที่กำลังเช็ก)

การ Persist และ Restart

  • เพิ่มฟิลด์ 2 ตัวใน bot_state.json: "trail_sl_frozen_side": "BUY" | "SELL" | null และ "entry_candle_frozen_side": "BUY" | "SELL" | null
  • เมื่อบอท restart ให้ restore ค่าเดิมของทั้ง 2 ฟิลด์
  • หลัง restore ถ้าสภาพ position/limit จริงทำให้เข้าเงื่อนไข reset (ไม่มี position + ไม่มี pending ทั้ง 2 ฝั่ง) ระบบจะ reset marker ทั้งคู่อัตโนมัติในรอบถัดไป
  • การ toggle Focus OFF ของฟีเจอร์ใด เคลียร์ marker เฉพาะของฟีเจอร์นั้น

สรุปโมเดลใหม่:
• freeze เป็น สถานะระดับฝั่ง (side-level) ไม่ใช่ per-ticket — และ แยก marker คนละตัว ต่อฟีเจอร์
• Trail SL กับ ECM ทำงานอิสระ: config คนละชุด, marker คนละตัว, toggle คนละปุ่ม
• threshold + TF เป็น gate สำหรับ "ฝั่งตรงข้ามจะได้ trail/ECM ตอนไหน" ตาม config ของฟีเจอร์นั้น ๆ
